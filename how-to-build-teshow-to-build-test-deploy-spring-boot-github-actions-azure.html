How to Build, Test, and Deploy a Spring Boot Application with GitHub Actions and Azure

Anindra February 22, 2022

GitHub Actions is a powerful tool that enables continuous integration and continuous deployment (CI/CD) workflows directly in your GitHub repository. In this tutorial, we will show you how to build, test, and deploy a Spring Boot application using Maven in a GitHub Action, and then deploy the application to an Azure Ubuntu server if the tests are successful.

Prerequisites

  • A GitHub account
  • A Spring Boot application with Maven
  • An Azure account with an Ubuntu server
  • The Azure CLI installed on your local machine

Step 1: Create a GitHub Action

To get started, create a new GitHub Action by adding a new YAML file to your repository's .github/workflows directory. For example, you could create a file named build.yml.

Copy the following YAML code into the file:

name: Java CI with Maven and Azure

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up JDK 11
      uses: actions/setup-java@v2
      with:
        java-version: 11

    - name: Cache Maven packages
      uses: actions/cache@v2
      with:
        path: ~/.m2/repository
        key: ${{ runner.os }}-maven-${{ hashFiles('**/pom.xml') }}

    - name: Build with Maven
      run: mvn package

    - name: Run tests with Maven
      run: mvn test

    - name: Deploy to Azure Ubuntu
      if: success()
      uses: azure/login@v1
      with:
        creds: ${{ secrets.AZURE_CREDENTIALS }}
      env:
        AZURE_WEBAPP_NAME: my-spring-boot-app
        AZURE_WEBAPP_PACKAGE_PATH: target/*.jar
        AZURE_WEBAPP_SLOT_NAME: production
      run: |
        az webapp deploy \
          --name $AZURE_WEBAPP_NAME \
          --slot $AZURE_WEBAPP_SLOT_NAME \
          --type java \
          --path $AZURE_WEBAPP_PACKAGE_PATH \
          --subscription ${{ secrets.AZURE_SUBSCRIPTION_ID }} \
          --resource-group ${{ secrets.AZURE_RESOURCE_GROUP }}

This YAML file defines a job named build that runs on the ubuntu-latest operating system. The job includes six steps:

  1. Checkout code: This step uses the actions/checkout action to clone the repository.
  2. Set up JDK 11: This step uses the actions/setup-java action to set up the Java environment. In this case, it sets up JDK 11.
  3. Cache Maven packages: This step uses the actions/cache action to cache the Maven repository to speed up subsequent builds. The cache key is based on the operating system and the hash of the pom.xml file.
  4. Build with Maven and Run tests with Maven: These steps use the run keyword to execute the mvn package and mvn test commands, respectively.
